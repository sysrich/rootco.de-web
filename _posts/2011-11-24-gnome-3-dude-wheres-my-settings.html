I'm a big fan of GNOME 3, but it does frustrate me to find settings I used to rely on suddenly missing from the standard configuration tools like System Settings

When testing openSUSE 12.1 I spent a lot of my time testing power, shutdown and suspend related issues

In previous versions, the default behaviour when pressing the hardware power button had been to show the user the power down menu with Reset, Shutdown, etc as options.

When upgrading a machine from earlier openSUSE versions I noticed that this behaviour was kept, but fresh installs used a new default, Suspend

seriously? Do I really need to have lid close, keyboard suspend button, my hardware power button, and the only visible power option in the default status menu all for suspending?
(Note: in openSUSE 12.1 we use the alternate status menu shell extension to make Shutdown and Hibernate options easier to find)

In earlier GNOMEs all I had to do to change the behaviour of the power button was fire up System Settings > Power

But in GNOME 3.2 there is a lovely blank grey area where that lovely setting used to be....

[caption id="attachment_25" align="alignnone" width="677" caption="You left a hole where my options should be"][/caption]

As my experience with upgrades shows, GNOME still has the settings, they're just hidden away in dconf, GNOMEs 'registry-like' configuration system.

The power button settings are hidden away under the dconf keyÂ /org/gnome/settings-daemon/plugins/power/button-power which you can change with the following command

gsettings set org.gnome.settings-daemon.plugins.power button-power 'interactive'

If you're like me and would like to fine tune your machine while still enjoying all the cool stuff in GNOME 3 I can recommend you install dconf-editor, which makes it pretty easy to explore the many settings hidden away that can help you get your GNOME session 'just right'

If you're an openSUSE user, installing is as complicated as

zypper in dconf-editor

[caption id="attachment_26" align="alignnone" width="802" caption="Ahh, there you are.."][/caption]

Please, upstream developers, you're doing wonderful beautiful things in GNOME 3, and making it simpler is great, but please don't forget we need an operating system to help put us in control of our hardware, not the other way around..
